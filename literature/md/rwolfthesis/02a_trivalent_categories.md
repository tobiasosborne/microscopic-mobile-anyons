# Chapter 2.1: Trivalent Categories

## Overview

Trivalent categories are monoidal categories generated by a **rotationally invariant trivalent vertex** τ ∈ Hom(1, X⊗X⊗X). They have simplified graphical calculus with additional rules.

**Key property**: Not necessarily fusion categories or MTCs, but some examples are both.

The Haagerup category H₃ has a trivalent description, enabling simpler calculations.

## Basic Definitions

### Definition: Evaluable Category

A k-linear category C is **evaluable** if dim(Hom(1,1)) = 1.

Hom(1,1) can be identified with ground field k by sending empty diagram to 1.

### Definition: Nondegenerate Category

A pivotal category C is **nondegenerate** if for every f ∈ Hom(X,Y) there exists f' ∈ Hom(Y,X) such that Tr(f'∘f) ≠ 0.

### Notation

For pivotal category C and object X:
```
C_k ≡ Hom_C(1, X^{⊗k})
```

Convention: X^{⊗0} = 1

### Definition: Trivalent Category

A **trivalent category** is a tuple (C, X, τ) where:
- C is nondegenerate evaluable pivotal category over C
- X is an object with:
  - dim(C_1) = 0 (no maps from X to unit)
  - dim(C_2) = 1 (X is simple)
  - dim(C_3) = 1 (unique trivalent vertex up to scalar)
- τ ∈ C_3 is the **trivalent vertex** (rotationally invariant)
- Category is generated by τ as a pivotal category

### Interpretation of Constraints

1. **dim(C_1) = 0**: X is not the unit and doesn't contain the unit
2. **dim(C_2) = 1**: X is simple (not isomorphic to unit, subobjects are 0 or X)
3. **dim(C_3) = 1**: No orientation label needed on trivalent vertex

### Properties

- X is **symmetrically self-dual** → can omit string orientations
- Every trivalent category is **spherical**
- Unoriented planar trivalent diagrams with n boundary points are elements of C_n

## Graphical Calculus

### Parameters d, b, t

Three characteristic parameters define a trivalent category:

**1. Loop relation** (from dim(C_0) = 1, evaluable):
```
○ = d
```
Parameter d is the dimension of X (must be nonzero).

**2. Lollipop relation** (from dim(C_1) = 0):
```
○─ = 0
```
Any diagram with a lollipop evaluates to zero.

**3. Bigon relation** (from dim(C_2) = 1):
```
╭─╮
│ │ = b · │
╰─╯
```
Parameter b is nonzero (bigon is nonzero pairing).

**4. Triangle relation** (from dim(C_3) = 1):
```
  ╱╲
 ╱  ╲
╱────╲ = t · Y
  │
```
Parameter t can be zero.

### Inner Product

Bilinear inner product of two trivalent graphs f, g with n boundary points:
```
⟨f, g⟩ = [f and g connected by n arcs at top]
```
Result is an element of C (ground field).

## Example: Fibonacci Category

The **Fibonacci category** Fib is both a trivalent category and a modular tensor category.

### Structure
- Simple objects: {1, X}
- Fusion rule: X ⊗ X = 1 + X
- Trivalent vertex: τ ∈ Hom(1, X⊗X⊗X)

### Parameters
- d² = d + 1 → d = φ = (1+√5)/2 (golden ratio) or conjugate (1-√5)/2
- b = 1
- t = (d-2)/(d-1)

### Morphism Space Dimensions (Fibonacci sequence!)
| n | dim(C_n) |
|---|----------|
| 0 | 1 |
| 1 | 0 |
| 2 | 1 |
| 3 | 1 |
| 4 | 2 |
| 5 | 3 |
| 6 | 5 |
| ... | ... |

### C_4 Relations

Four diagrams without internal faces span C_4:
- w₁: Two parallel arcs (left-right pairing)
- w₂: Two parallel arcs (top-bottom pairing)
- w₃: Y-tree left
- w₄: Y-tree right

For Fibonacci (dim(C_4) = 2), two relations:
```
w₄ - w₃ + (1/(d+1))w₁ + (1/(d-1))w₂ = 0
w₁ - (1/d)w₂ = w₃
```

## Cubic Trivalent Categories

### Definition: Cubic Category

A trivalent category is **cubic** if dim(C_4) = 4.

The four diagrams w₁, w₂, w₃, w₄ form a basis (no relations among them).

### Square Decomposition

For cubic categories, diagrams with internal faces must be decomposed. The **square diagram**:
```
□ with four external legs
```

can be written as:
```
□ = α(w₁ + w₂) + β(w₃ + w₄)
```

where:
- α = b(b² + bt - t²) / (bd + t + dt)
- β = (t²(d+1) - b²) / (bd + t + dt)

This allows evaluation of any diagram with square faces.

## The Trivalent Category H₃

### Overview

The Haagerup H₃ category is a **cubic trivalent category** (dim(C_4) = 4).

### Parameters
- d = (3 + √13)/2 ≈ 3.303
- b = 1
- t = -(2/3)d + (5/3) = -(√13)/3

### Morphism Space Dimensions
- dim(C_4) = 4
- dim(C_5) = 11
- dim(C_6) = 37

### C_5 Basis (11 diagrams)

10 diagrams without faces:
- 5 rotations of "chain" pattern
- 5 rotations of "star with bigon" pattern

Plus 1 pentagon:
- Regular pentagon with 5 external legs

### C_6 Basis (37 diagrams)

34 diagrams without faces (various patterns with rotations):
- 3-arc patterns
- Crossing patterns
- Y-vertex patterns
- Double-Y patterns

7 diagrams with one face:
- Regular hexagon
- 6 rotations of "pentafork" (pentagon with one bigon)

### Orthonormal Basis

Finding orthonormal basis via Gram-Schmidt becomes infeasible for large morphism spaces. The **fusion basis** from full category provides canonical orthonormal basis.

### Connection to Full Category

- Trivalent H₃ is a **subcategory** of full H₃
- Full category recoverable via **Karoubi envelope** (idempotent completion)
- Trivalent calculus gives information about parts of full category

## Applications

### Seed Data for F-Symbols

Trivalent relations provide equations for F-symbols:
- Interpret trivalent diagrams as morphisms in full category
- Apply F-moves to simplify
- Compare with trivalent relations
- Extract constraints on F-symbol entries

### Example: Square to F-Symbol Relation

For H₃, interpreting the square diagram in terms of the full category:
```
(F_ρ^{ρρρ})†_{_αρ,ρ} (F_{_αρ}^{ρρρ})_{ρρ} √d_ρ = α(F_ρ^{ρρρ})†_{_αρ,1} + β(F_ρ^{ρρρ})†_{_αρ,ρ}
```

Similar relations for other F-symbol entries.
